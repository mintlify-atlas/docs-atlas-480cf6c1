## Selecting a Style Guide

Choose your style guide from the **sidebar** under "Base Style Guide":

1. Open the sidebar (if closed)
2. Locate the **Base Style Guide** dropdown
3. Select: `Google`, `Microsoft`, or `RedHat`
4. Run linting to apply the selected rules

The selected guide determines which ruleset Vale uses to check your content.

<Info>
  You can change style guides at any time and re-run linting to see how your content performs against different standards.
</Info>

## Google Developer

### Overview

The [Google Developer Documentation Style Guide](https://developers.google.com/style) provides comprehensive guidelines for technical documentation, with emphasis on:

- API documentation clarity
- Developer-focused content
- Technical accuracy and precision
- Accessible language for international audiences

### Key Characteristics

<AccordionGroup>
  <Accordion title="Tone & Voice">
    - **Conversational but professional**
    - Avoids first-person pronouns (`I`, `my`, `me`)
    - Uses second-person (`you`) to address readers
    - Maintains objectivity in technical explanations
  </Accordion>
  
  <Accordion title="Formatting Standards">
    - Time format: `10 AM` (uppercase with space)
    - Date format: `July 12, 2026` (Month Day, Year)
    - Avoids Latin abbreviations (`e.g.`, `i.e.`)
    - Uses em dashes (â€”) with no surrounding spaces
    - Exclamation marks discouraged
  </Accordion>
  
  <Accordion title="Language Guidelines">
    - Spells out acronyms on first use
    - Avoids contractions in formal contexts
    - Gender-neutral language required
    - Consistent heading capitalization (sentence case)
    - No optional plurals (e.g., "file(s)")
  </Accordion>
</AccordionGroup>

## Microsoft Writing

### Overview

The [Microsoft Writing Style Guide](https://learn.microsoft.com/style-guide) focuses on making technical content accessible to diverse audiences while maintaining professional standards.

### Key Characteristics

<AccordionGroup>
  <Accordion title="Tone & Voice">
    - **Warm and approachable**
    - Emphasizes customer perspective
    - Allows selective use of contractions
    - Encourages clarity over formality
  </Accordion>
  
  <Accordion title="Formatting Standards">
    - Uses serial commas (Oxford commas)
    - Prefers active voice over passive
    - Consistent capitalization in headings
    - Clear bullet point structure
  </Accordion>
  
  <Accordion title="Language Guidelines">
    - Inclusive language required
    - Avoids jargon when possible
    - Uses simple words over complex ones
    - Respects cultural and regional differences
  </Accordion>
</AccordionGroup>

## Combining with Custom Rules

You can **layer custom rules** on top of any base style guide:

1. Select your base guide (`Google`, `Microsoft`, or `RedHat`)
2. Upload custom rules in the sidebar (see [Custom Rules](/user-guide/custom-rules))
3. Run linting to apply **both** base and custom rules

This allows you to:
- Extend style guides with project-specific terms
- Add organization-specific rules
- Enforce brand guidelines alongside industry standards

<Tip>
  Custom rules augment the base style guide rather than replacing it. Both sets of rules are checked during linting.
</Tip>
